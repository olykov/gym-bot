# UI/UX Review — `apps/web` (Gym Mini App)

> Дата: 2026-06-11. Скоуп: клиентский Mini App `apps/web` (Dashboard, Progress, History,
> Record flow, shell). Ревью против frontend-spec.md, мобильных best practices и
> Telegram Mini App guidelines. Статус: **approved 2026-06-12**; задачи —
> `tasks/ux-polish/` (GYM-116…GYM-123).

## 0. Что уже сильно (короткo, но честно)

Дизайн-система реально дисциплинированная: один shell, токены, Chalk & Iron выдержан,
skeleton-states везде, `prefers-reduced-motion` повсюду, haptics, safe-areas (включая
fullscreen Bot API 8.0), BackButton ownership продуман, tap-count budget записи сета
выполнен. Это уровень выше среднего продакшена. Ревью ниже — про то, что добавит
следующий уровень полировки, а не про «всё переделать».

---

## 1. Критичное (P0) — реальные UX-дефекты

### 1.1 Скролл-позиция теряется при возврате (History → день → назад)
`AppShell` рендерит `<Container revealKey={location.pathname}>`, а `Container` ставит
`key={revealKey}` на `<main>`. Любая навигация **ремаунтит** скролл-контейнер: вернулся
из `/history/:date` в `/history` — список снова сверху, и reveal-stagger проигрывается
заново. Для списка на 12+ недель это болезненно.
- Фикс: не ремаунтить `<main>`; играть reveal только на push-навигации (или только на
  первом маунте route), восстанавливать scroll per-route (sessionStorage map или
  `ScrollRestoration`-подход react-router).

### 1.2 Тултипы Activity-грида не работают на тач-устройствах
`Cell` использует `title={cellTooltip(cell)}` — на телефоне `title` не показывается
никогда, а приложение на 99.9% телефонное. Сейчас юзер не может узнать «сколько сетов
было в этот день».
- Фикс: tap по ячейке → выбранная ячейка + строка под гридом «12 sets · MON 02 JUN»
  (и/или переход в `/history/:date`, если день есть в истории — это ещё и навигация).

### 1.3 Empty-states отправляют в бота, хотя запись уже в апке
Dashboard/Progress/History: «Log a workout **in the bot** and …». Record flow в Mini App
уже первоклассный — тексты противоречат продукту и прячут главный CTA.
- Фикс: «Tap **+** to log your first set» + кнопка-action в `EmptyState` (проп `action`
  уже есть, не используется), открывающая RecordSheet. Это же онбординг FAB'а.

### 1.4 Вложенные шиты и BackButton — проверить двойное закрытие
`ManageSheet` (zIndex 40) открывается поверх записи/пикера, оба — `BottomSheet`, и оба
в открытом состоянии вешают свой `BackButton.onClick`. SDK вызывает **всех** подписчиков:
один Back, похоже, закроет оба шита (или закроет manage + шагнёт пикер назад).
- Фикс: стек back-хендлеров (один владелец, верхний шит перехватывает), либо

  `wireBackButton` с приоритетом/единственным активным подписчиком. Сначала —
  воспроизвести на устройстве.

## 2. Высокий приоритет (P1) — момент записи сета «мёртвый»

Это подтверждённая боль оператора: «кнопка остаётся на месте, строка просто появляется».
Полная концепция анимаций/геймификации — в `03-progressive-overload-concept.md`; здесь
только UI-механика (всё за `prefers-reduced-motion`, токен-easing `--ease-out-soft`):

1. **Появление строки в recap** — не мгновенная вставка, а 240ms fade + rise 8px (тот же
   язык, что `.reveal`) + одноразовый `pr-flare`-подобный фон-флэш на новой строке
   (не только при PR). Сейчас flare есть ТОЛЬКО на PR-beat.
2. **Морф кнопки Save** — после успеха кнопка на ~600ms становится «✓ SET 2 SAVED»
   (без layout-shift, только контент + лёгкий scale 1.02→1), затем «SAVE SET 3».
   Сейчас label меняется мгновенно — мозг не регистрирует событие.
3. **Тик заголовка SET N** — цифра «прокатывается» вверх (translateY swap, 180ms).
4. **Счётчик прогресса сессии** в шапке панели: «3 sets · 2 exercises today» — живое
   ощущение накопления.

## 3. P1 — навигационные переходы и жесты

- **Drill-in без направления.** `/history` → `/history/:date` — просто re-stagger.
  Мобильный паттерн: push slide-left на вход, slide-right на выход (View Transitions API
  деградирует в instant — идеально ложится на reduced-motion правило). Слайд уже есть
  внутри RecordPicker — распространить язык на роуты.
- **BottomSheet не закрывается свайпом вниз.** Grab-handle чисто декоративный —
  это нарушенное ожидание: ручка = «можно тянуть». Добавить drag-to-dismiss
  (порог ~30% высоты / velocity), scrim-fade пропорционально смещению.
  Для `fixedHeight` (record) можно оставить только handle-зону как drag-target,
  чтобы не конфликтовать с внутренним скроллом.
- **Swipe-back с дня истории** (правый край → назад) — опционально, Telegram iOS
  частично перехватывает это сам; проверить на устройстве прежде чем строить.

## 4. P2 — полировка по экранам

### Record / SetLogger
- `Set n ✓`-строки (бот-сеты до загрузки day-detail) выглядят как баг рядом с
  `100kg × 8`. Минимум — выровнять стиль (`— · —` в той же типографике), максимум —
  дождаться serverSets (они почти всегда уже в кэше).
- Stepper: нет long-press repeat на ±. Для «со 100 на 60» это 16 тапов. Hold-to-repeat
  с ускорением (250ms → 80ms) — дёшево и резко улучшает.
- Шаг веса жёстко 2.5. Для гантелей/тросов часто нужен шаг 1–2. Не строить настройки
  (YAGNI), но вынести в константу и обдумать «умный шаг» от истории упражнения (v2).
- «Done» закрывает шит молча. Сессия заканчивается без финального аккорда — см. док 03
  (session summary). Даже без геймификации: лёгкий success-haptic на Done уместен.
- Ошибка `POST /training` 409 (дубль номера сета) показывается тем же generic
  «Couldn't save…» — стоит различить (см. tech-review §5).

### Dashboard
- `StatCard` имеет `press-95`, но не кликабелен — ложный аффорданс. Либо убрать, либо
  сделать карточки входами: Sets → History, PRs → список PR (в бэклог), Streak →
  объяснение правила стрика (юзеру неочевидно, что такое week streak).
- Activity-грид: ось месяцев отсутствует — 26 недель без подписей времени читаются хуже,
  чем у GitHub. Лёгкая строка меток месяцев сверху сильно поможет ориентироваться.

### Progress
- Auto-select топ-упражнения — отлично. Но «By Set» с 5+ сетами на 360px шумный;
  legend скроллится — ок, но рассмотреть cap в 4 линии + «show all sets».
- При смене упражнения данные графика меняются без перехода — 200ms fade/morph
  (ECharts `animation` уже умеет) сгладит.
- Тултип ECharts на таче требует tap-and-hold — проверить, что обычный tap по точке
  показывает значение (triggerOn: 'click' как fallback на мобиле).

### History
- `DayCard` не показывает объём/PR дня — добавить тонкие сигналы значимости дня
  (например, accent-точка «PR day»). Связано с доком 03.
- «That's the beginning.» — милая деталь, оставить :)

### Shell
- Заголовок `AppHeader` всегда — статический tab label; на Dashboard место можно
  использовать живее («THU · 11 JUN» / streak). Низкий приоритет.
- `ErrorState` использует «❌» эмодзи — выбивается из Chalk & Iron (иконки в апке —
  только token-stroked SVG). Заменить на инлайн-SVG.
- AuthGate: «Loading / Signing you in…» как EmptyState-заголовок выглядит сыро;
  скелет-шелл без слов был бы чище.

## 5. P2 — доступность / i18n / темы

- **i18n — фактически UX-дефект**: вся UI — английские хардкоды, аудитория
  Telegram-бота во многом русскоязычная. Инфраструктура (`getLocale()`, en/ru) уже
  есть, каталога нет (GYM-109). Это пункт №1 в tech-review, но эффект — юзерский.
- **Focus trap в BottomSheet заявлен в докстринге, но не реализован** (только focus на
  панель при открытии). Для Telegram-WebView некритично, но Tab на desktop уводит фокус
  под scrim. Реализовать примитивный trap или убрать заявление.
- Контраст: `--hint` (#8a8f99) на `--secondary-bg` для label-текста на тайлах-skeleton
  и подписи Stepper — на грани 4.5:1; прогнать палитру обеих тем через контраст-чек
  (спека сама требует «verify in dark» в 4 местах — собрать это в один QA-чеклист
  и закрыть).
- `aria-live` на recap-список SetLogger (новая строка озвучивается) — мелочь, дёшево.
